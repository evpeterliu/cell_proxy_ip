## 📖 项目概述

### 项目目标

构建一个**基于手机卡流量的 SOCKS5 代理服务平台**，让用户可以：
- 使用真实的手机卡 IP 地址访问互联网
- 获得动态住宅 IP（每次重连 IP 可能变化）
- 绕过某些网站的 IP 限制

### 核心价值

- **对用户：** 提供高质量的住宅 IP 代理服务
- **对平台：** 利用闲置手机卡流量变现
- **对终端设备主：** 分享手机卡流量获得收益

## 🔧 核心问题与解决方案

### 问题 1：NAT 后的设备如何对外提供服务？

**问题描述：**
- 手机卡设备在运营商的 NAT 网络后面
- 没有公网 IP 地址
- 外部用户无法直接连接

**解决方案：反向连接 + 中继服务器**

```
外部用户 → 公网服务器 → NAT 后的手机卡设备
         (中转)      (反向连接)
```

**通俗解释：**
就像你在家里（NAT 后），外面的人无法直接敲你家门。但你可以主动打电话（反向连接）给中介公司（公网服务器），中介公司接到客户后，通过你打来的电话线转接给你。

---

### 问题 2：所有流量都要经过公网服务器吗？

**答案：是的，必须经过**

**原因：**
1. 手机网络是运营商级 NAT（CGNAT），无法配置端口映射
2. 外部用户无法直接连接到 NAT 后的设备
3. 必须通过公网服务器中转所有数据

**能否优化：**
- ✅ 可以尝试 UDP 打洞技术，成功率约 15-20%
- ✅ 成功后可以直连，节省带宽
- ❌ 但 80-85% 的情况仍需中继

---

### 问题 3：如何降低公网服务器成本？

**方案对比：**

| 方案 | 带宽消耗 | 成本 | 说明 |
|------|---------|------|------|
| **国内服务器** | 100% 中转 | ¥30,000/月 | 带宽贵 |
| **国外服务器** | 100% 中转 | ¥1,700/月 | 带宽便宜 |
| **智能混合** | 80% 中转 | ¥1,400/月 | 部分直连 |

**推荐：使用国外服务器（Hetzner/OVH）+ 智能混合方案**

    │
    ▼

    │
    ▼
8. 公网服务器返回数据给用户
    │
    ▼

### 🆕 新增需求：SIM卡信号监控与自动切换

**背景：**
- 在菲律宾部分地区信号不稳定
- Net终端（RV1103设备）信号比手机更弱
- 手机卡可能注册不到网络

**核心需求：**

**1. SIM卡信号监控**
- [ ] 实时监控SIM卡信号强度（RSSI/RSRP）
- [ ] 监控网络注册状态（已注册/未注册/搜索中）
- [ ] 记录信号历史数据
- [ ] 信号弱时告警（< -100dBm）

**2. SIM卡自动切换机制**
- [ ] SIM1注册网络失败后，自动切换到SIM2
- [ ] 检测SIM2是否能注册网络
- [ ] 如果SIM2也失败，切回SIM1重试
- [ ] 记录切换日志

**3. 切换策略**
```
SIM1尝试注册网络
    │
    ▼
注册失败（超时30秒）
    │
    ▼
自动切换到SIM2
    │
    ▼
SIM2尝试注册网络
    │
    ├─ 成功 → 使用SIM2
    │
    └─ 失败 → 切回SIM1，等待60秒后重试
```

**4. 技术实现要点**
- 使用EC600M AT命令查询信号强度：`AT+CSQ`
- 查询网络注册状态：`AT+CREG?`
- 切换SIM卡：`AT+QDSIM=<sim_id>`（0=SIM1, 1=SIM2）
- 注册超时时间：30秒
- 重试间隔：60秒

**5. 监控指标**
- SIM1/SIM2信号强度（dBm）
- 当前使用的SIM卡
- 网络注册状态
- 切换次数统计
- 切换成功率

**6. 告警规则**
```
1. 信号强度 < -100dBm → 预警
2. 连续3次切换失败 → 告警
3. 设备超过5分钟无网络 → 紧急告警
4. 单台设备24小时切换次数 > 20次 → 异常告警
```

## 🔄 技术方案对比

### 方案 A：纯 TCP 中继（推荐新手）

**优点：**
- ✅ 实现简单，使用现成工具（frp）
- ✅ 稳定性高，100% 可用
- ✅ 维护成本低

**缺点：**
- ❌ 延迟较高（+50ms）
- ❌ 所有流量经过服务器

**适用场景：**
- 初期快速上线
- 技术团队经验不足
- 用户对延迟不敏感

**成本：** ¥1,700/月（100 台设备）

---

### 方案 B：智能混合（推荐）

**优点：**
- ✅ 延迟更低（-40%）
- ✅ 节省带宽成本（-16%）
- ✅ 用户体验更好

**缺点：**
- ⚠️ 开发复杂度高
- ⚠️ 需要 NAT 穿透技术
- ⚠️ 调试难度大

**适用场景：**
- 有技术实力的团队
- 追求极致性能
- 规模化运营

**成本：** ¥1,400/月（100 台设备）

---
