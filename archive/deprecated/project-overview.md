## 🎯 项目目标

**核心痛点：** 手机卡断网后无法远程排查问题

**解决方案：** 通过中转服务器 + 双网卡路由 + MQTT远程管理，实现稳定的IP代理服务和完善的远程故障排查能力

**成功标准：**
- ✅ 10台设备在菲律宾稳定运行
- ✅ 手机卡断网时，30分钟内远程诊断
- ✅ 设备在线率 >95%

## 🏗️ 技术架构

### 系统架构图

```
用户 → 香港中转服务器 → frp隧道 → net终端(手机卡) → 目标网站
                                    ↓
                          MQTT调度服务(Go + PostgreSQL)
                                    ↓
                              监控面板(Grafana)
```

### 技术栈

**设备端（RV1103）：**
- Ubuntu 22.04 ARM64
- Python 3.11（网络监控、MQTT客户端）
- tiny-frpc（反向隧道）
- microsocks（SOCKS5代理）

**服务端：**
- Go + PostgreSQL（MQTT调度服务）
- Mosquitto（MQTT Broker）
- frps（反向代理服务器）
- gost（SOCKS5代理）
- Grafana + Prometheus（监控）
