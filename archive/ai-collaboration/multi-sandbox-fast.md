## 🤖 多沙盒协作机制

### 1. 任务分配（自动化）

**任务队列系统：**
```python
# task_queue.py
# 运行在开发服务器上

class TaskQueue:
    def __init__(self):
        self.queue = []  # 待分配任务
        self.sandboxes = {
            'sandbox-1': {'status': 'idle', 'current_task': None},
            'sandbox-2': {'status': 'idle', 'current_task': None},
            'sandbox-3': {'status': 'idle', 'current_task': None},
            'sandbox-4': {'status': 'idle', 'current_task': None},
        }
    
    def assign_task(self):
        """自动分配任务给空闲的沙盒"""
        for sandbox_id, info in self.sandboxes.items():
            if info['status'] == 'idle' and self.queue:
                task = self.queue.pop(0)
                info['status'] = 'busy'
                info['current_task'] = task
                # 通过GitHub Issue通知沙盒
                self.notify_sandbox(sandbox_id, task)
```

### 3. 测试设备分配

**设备管理系统：**
```python
# device_manager.py

class DeviceManager:
    def __init__(self):
        self.devices = {
            'device-1': {'status': 'idle', 'assigned_to': None},
            'device-2': {'status': 'idle', 'assigned_to': None},
            'device-3': {'status': 'reserved', 'assigned_to': 'integration'},
        }
    
    def request_device(self, sandbox_id):
        """沙盒请求测试设备"""
        for device_id, info in self.devices.items():
            if info['status'] == 'idle':
                info['status'] = 'busy'
                info['assigned_to'] = sandbox_id
                return device_id
        return None  # 无可用设备
    
    def release_device(self, device_id):
        """释放设备"""
        self.devices[device_id]['status'] = 'idle'
        self.devices[device_id]['assigned_to'] = None
```

## 🔧 内网编译服务器暴露方案

### 推荐：GitHub Actions自托管runner ⭐

**为什么？**
- ✅ 安全（不需要暴露端口）
- ✅ 集成度高（直接在GitHub Actions中使用）
- ✅ 自动化（代码提交自动触发）
- ✅ 可扩展（可以添加多个runner）

**架构：**
```
GitHub
    ↓ Webhook
GitHub Actions
    ↓ 分配任务
内网编译服务器（Self-hosted Runner）
    ↓ 编译
产出上传到GitHub Artifacts
    ↓ 下载
AI沙盒 / 测试服务器
```

**配置步骤：**
```bash
# 在内网编译服务器上
cd /opt/actions-runner
./config.sh --url https://github.com/evpeterliu/cell_proxy_ip \
  --token YOUR_TOKEN \
  --name rv1103-builder \
  --labels rv1103,arm64,builder

./run.sh
```

**GitHub Actions配置：**
```yaml
# .github/workflows/build-rv1103.yml
name: Build RV1103

on: [push, pull_request]

jobs:
  build:
    runs-on: [self-hosted, rv1103, builder]
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Cross compile for RV1103
        run: |
          GOOS=linux GOARCH=arm64 go build -o bin/mqtt-client cmd/main.go
      
      - name: Upload artifact
        uses: actions/upload-artifact@v3
        with:
          name: mqtt-client-arm64
          path: bin/mqtt-client
```

## ⚡ 极速开发的关键

### 1. 并行最大化

**传统串行：**
```
T0.1 → T0.2 → T0.3 → T0.4 → T1.1 → ...
总耗时：30天
```

**极速并行：**
```
Day 1-5:
  Sandbox-1: T0.4 + T1.3 + T1.4
  Sandbox-2: T0.1 + T1.1 + T1.2
  Sandbox-3: T0.2
  Sandbox-4: T0.3 + T1.5

总耗时：5天（节省25天）
```

### 2. 自动化一切

**自动化清单：**
- ✅ 代码提交 → 自动编译
- ✅ 编译成功 → 自动部署
- ✅ 部署成功 → 自动测试
- ✅ 测试通过 → 自动通知
- ✅ 任务完成 → 自动分配下一个

**人工只做：**
- 验收
- 决策
- 风险处理

### 3. 资源充足

**不考虑成本的优势：**
- 6台服务器 → 无资源瓶颈
- 4个沙盒 → 最大并行度
- 3台测试设备 → 无需排队
