## 🛠️ 技术栈选型

### 后端技术栈

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| **代理服务器** | Go + gost | 高性能，原生支持 SOCKS5 |
| **API 服务** | Python FastAPI | 开发效率高，异步支持好 |
| **反向代理** | frp | 成熟稳定，社区活跃 |
| **数据库** | PostgreSQL 15 | 可靠性高，支持 JSON |
| **缓存** | Redis 7 | 高性能，支持多种数据结构 |
| **消息队列** | Redis Streams | 轻量级，无需额外组件 |
| **监控** | Prometheus + Grafana | 业界标准 |

### 前端技术栈

| 组件 | 技术选型 |
|------|---------|
| **框架** | Vue 3 + TypeScript |
| **UI 库** | Element Plus |
| **状态管理** | Pinia |
| **HTTP 客户端** | Axios |
| **图表** | ECharts |

### 设备端技术栈

| 组件 | 技术选型 |
|------|---------|
| **客户端** | Python 3.11 |
| **反向连接** | frpc |
| **进程管理** | systemd |
| **日志** | logrotate |

## 🔄 数据流设计

### 用户请求完整流程

```
步骤 1: 用户发起代理请求
┌──────────────────────────────────────────────────────┐
│ 用户软件（如浏览器）                                   │
│ 配置: SOCKS5 proxy.example.com:1080                  │
│ 认证: username / password                            │
└──────────────────────────────────────────────────────┘
                    │
                    │ TCP 连接
                    ▼
步骤 2: Nginx 负载均衡
┌──────────────────────────────────────────────────────┐
│ Nginx (端口 1080)                                     │
│ 功能:                                                 │
│ - SSL/TLS 终止                                        │
│ - 负载均衡（轮询）                                     │
│ - 限流保护                                            │
└──────────────────────────────────────────────────────┘
                    │
                    │ 转发到后端
                    ▼
步骤 3: SOCKS5 代理服务器
┌──────────────────────────────────────────────────────┐
│ gost SOCKS5 服务器                                    │
│ 功能:                                                 │
│ 1. 解析 SOCKS5 协议                                   │
│ 2. 验证用户名密码（查询 Redis）                        │
│ 3. 检查用户套餐和余额                                  │
│ 4. 选择可用的手机卡设备                                │
└──────────────────────────────────────────────────────┘
                    │
                    │ 通过 frp 隧道
                    ▼
步骤 4: frps 转发
┌──────────────────────────────────────────────────────┐
│ frps 服务器                                           │
│ 功能:                                                 │
│ - 维护与所有 frpc 的连接                              │
│ - 根据设备 ID 转发请求                                │
│ - 流量统计                                            │
└──────────────────────────────────────────────────────┘
                    │
                    │ 反向隧道
                    ▼
步骤 5: 手机卡设备处理
┌──────────────────────────────────────────────────────┐
│ frpc 客户端 + gost                                    │
│ 功能:                                                 │
│ 1. 接收代理请求                                        │
│ 2. 使用手机卡流量访问目标网站                          │
│ 3. 返回响应数据                                        │
└──────────────────────────────────────────────────────┘
                    │
                    │ 手机卡网络
                    ▼
步骤 6: 访问目标网站
┌──────────────────────────────────────────────────────┐
│ 目标网站（如 taobao.com）                             │
│ 看到的源 IP: 手机卡的真实 IP                          │
└──────────────────────────────────────────────────────┘
                    │
                    │ 响应数据
                    ▼
步骤 7: 原路返回
数据按原路返回：
目标网站 → 手机卡设备 → frps → gost → Nginx → 用户
```

## 📊 数据库设计

### 核心表结构

**用户表 (users)**
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    email VARCHAR(100),
    phone VARCHAR(20),
    balance DECIMAL(10, 2) DEFAULT 0,
    status VARCHAR(20) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

**设备表 (devices)**
```sql
CREATE TABLE devices (
    id SERIAL PRIMARY KEY,
    device_id VARCHAR(50) UNIQUE NOT NULL,
    name VARCHAR(100),
    carrier VARCHAR(20),  -- 移动/联通/电信
    network_type VARCHAR(10),  -- 4G/5G
    location VARCHAR(100),  -- 地区
    status VARCHAR(20) DEFAULT 'offline',
    last_heartbeat TIMESTAMP,
    total_traffic BIGINT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW()
);
```

**会话表 (sessions)**
```sql
CREATE TABLE sessions (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    device_id INTEGER REFERENCES devices(id),
    start_time TIMESTAMP DEFAULT NOW(),
    end_time TIMESTAMP,
    bytes_sent BIGINT DEFAULT 0,
    bytes_received BIGINT DEFAULT 0,
    status VARCHAR(20) DEFAULT 'active'
);
```
