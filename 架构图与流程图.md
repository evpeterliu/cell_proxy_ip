## 📊 系统架构图

### 1. 整体架构（鸟瞰图）

```
                        ┌─────────────────────┐
                        │      互联网用户      │
                        │  (需要使用代理)      │
                        └──────────┬──────────┘
                                   │

                                   │    proxy.example.com:1080
                                   │
                        ┌──────────▼──────────┐
                        │   公网服务器集群     │
                        │  (德国 Hetzner)     │
                        │                     │
                        │  ┌───────────────┐ │
                        │  │ Nginx 负载均衡│ │
                        │  └───────┬───────┘ │
                        │          │         │
                        │  ┌───────▼───────┐ │
                        │  │ SOCKS5 服务器 │ │
                        │  │   (gost)      │ │
                        │  └───────┬───────┘ │
                        │          │         │
                        │  ┌───────▼───────┐ │
                        │  │ frps 中继服务 │ │
                        │  └───────┬───────┘ │
                        └──────────┼─────────┘
                                   │
                                   │ ② 反向隧道
                                   │    (设备主动连接)
                                   │
        ┌──────────────────────────┼──────────────────────────┐
        │                          │                          │
        │                          │                          │
┌───────▼────────┐      ┌──────────▼─────────┐      ┌────────▼───────┐
│ 手机卡设备 001  │      │ 手机卡设备 002      │      │ 手机卡设备 N    │
│ 树莓派 + 移动4G │      │ 树莓派 + 联通5G     │      │ 树莓派 + 电信4G │
│ 广东深圳       │      │ 北京               │      │ 上海           │
└───────┬────────┘      └──────────┬─────────┘      └────────┬───────┘
        │                          │                          │
        │                          │                          │
        └──────────────────────────┼──────────────────────────┘
                                   │
                                   │ ③ 使用手机卡流量访问
```

**说明：**
- ① 用户配置代理地址，所有流量通过代理
- ② 手机卡设备主动连接公网服务器（解决 NAT 问题）
- ③ 设备使用手机卡流量访问目标网站

## 🔄 数据流程图

### 1. 用户请求完整流程（详细版）

```
┌─────────┐
│ 步骤 1   │  用户发起请求
└────┬────┘
     │
     │  用户在浏览器中访问 taobao.com
     │  浏览器配置了 SOCKS5 代理：proxy.example.com:1080
     │
┌────▼────┐
│ 步骤 2   │  连接代理服务器
└────┬────┘
     │
     │  浏览器发送 SOCKS5 握手请求
     │  包含：用户名、密码
     │
┌────▼────┐
│ 步骤 3   │  Nginx 接收请求
└────┬────┘
     │
     │  Nginx 检查：
     │  - 连接数是否超限？
     │  - IP 是否在黑名单？
     │  通过后转发到 gost
     │
┌────▼────┐
│ 步骤 4   │  gost 验证用户
└────┬────┘
     │
     │  gost 查询 Redis：
     │  - 用户名密码是否正确？
     │  - 用户余额是否充足？
     │  - 用户状态是否正常？
     │
┌────▼────┐
│ 步骤 5   │  选择手机卡设备
└────┬────┘
     │
     │  gost 根据策略选择设备：
     │  - 负载最低的设备
     │  - 地区匹配（如果用户指定）
     │  - 运营商匹配（如果用户指定）
     │
┌────▼────┐
│ 步骤 6   │  通过 frps 转发
└────┬────┘
     │
     │  gost 将请求发送给 frps
     │  frps 查找对应设备的连接
     │  通过反向隧道转发给设备
     │
┌────▼────┐
│ 步骤 7   │  设备接收请求
└────┬────┘
     │
     │  frpc 接收到请求
     │  转发给本地 gost
     │  gost 解析目标地址：taobao.com:443
     │
┌────▼────┐
│ 步骤 8   │  使用手机卡访问
└────┬────┘
     │
     │  设备通过 USB 4G 模块
     │  使用手机卡流量
     │  访问 taobao.com
     │  源 IP：手机卡的真实 IP
     │
┌────▼────┐
│ 步骤 9   │  淘宝返回数据
└────┬────┘
     │
     │  淘宝服务器返回网页内容
     │  目标 IP：手机卡 IP
     │
┌────▼────┐
│ 步骤 10  │  原路返回
└────┬────┘
     │
     │  数据按原路返回：
     │  淘宝 → 设备 → frpc → frps → gost → Nginx → 用户
     │
┌────▼────┐
│ 步骤 11  │  用户收到数据
└────┬────┘
     │
     │  浏览器显示淘宝网页
     │  用户看到的内容和直接访问一样
     │
┌────▼────┐
│ 步骤 12  │  流量计费
└─────────┘
     
     API 服务记录本次会话：
     - 用户 ID
     - 使用的设备
     - 流量大小
     - 扣除费用
```

### 2. 设备心跳流程

```
┌─────────────────┐
│ 手机卡设备       │
│ (每 30 秒执行)   │
└────────┬────────┘
         │
         │ 1. 收集数据
         │    - CPU 使用率
         │    - 内存使用率
         │    - 网络流量
         │    - 温度
         │
┌────────▼────────┐
│ 监控脚本         │
│ (Python)        │
└────────┬────────┘
         │
         │ 2. 发送 HTTP POST 请求
         │    POST /api/devices/heartbeat
         │    {
         │      "device_id": "device_001",
         │      "cpu": 45.2,
         │      "memory": 62.8,
         │      "status": "online"
         │    }
         │
┌────────▼────────┐
│ API 服务器       │
└────────┬────────┘
         │
         │ 3. 更新数据库
         │    UPDATE devices
         │    SET status='online',
         │        last_heartbeat=NOW()
         │    WHERE device_id='device_001'
         │
┌────────▼────────┐
│ Redis 缓存       │
└────────┬────────┘
         │
         │ 4. 更新缓存
         │    SET device:001:status "online"
         │    EXPIRE device:001:status 90
         │
┌────────▼────────┐
│ 监控系统         │
│ (Prometheus)    │
└─────────────────┘
         
         5. 记录指标
            device_cpu{id="001"} 45.2
            device_memory{id="001"} 62.8
```
