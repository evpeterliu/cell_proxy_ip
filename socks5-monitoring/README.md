## 🎯 功能说明

### 监控对象

- **服务器：** codex.adusun.com (154.92.2.59)
- **端口范围：** 21880-21896（共17个端口）
- **设备数量：** 17台net终端（RV1103 + EC600M）
- **监控频率：** 按需/每小时/每天

### 监控指标

1. **端口在线状态**（可用/不可用）
2. **出口IP地址**（每个在线端口的实际出口IP）
3. **设备序列号映射**（端口→设备SN→SIM卡号）
4. **在线率统计**（可用设备/总设备）
5. **IP变化追踪**（对比历史记录）

## 🚀 快速开始

### 1. 执行快速检测

**前提条件：** 
- 有203.3.113.133服务器的SSH访问权限
- SSH密钥：`~/.ssh/server_key`

**方法一：一行命令检测**
```bash
ssh -i ~/.ssh/server_key root@203.3.113.133 'for port in {21880..21896}; do echo -n "端口 $port: "; ip=$(timeout 5 curl -x socks5://ppp:ppp@codex.adusun.com:$port -s https://ip.sb 2>/dev/null); if [ -n "$ip" ]; then echo "✅ $ip"; else echo "❌ 不可用"; fi; sleep 0.5; done'
```

**方法二：使用Python脚本**
```bash
# 1. SSH到中转服务器
ssh -i ~/.ssh/server_key root@203.3.113.133

# 2. 上传并运行脚本
python3 quick_socks5_check.py
```

### 2. 查看最新报告

```bash
cat reports/$(ls -t reports/ | head -1)
```

### 3. 查看设备映射

```bash
# 查看端口→设备→SIM卡映射关系
grep "^|" reports/$(ls -t reports/ | grep device_mapping | head -1)
```

## 🛠️ 脚本使用

### quick_socks5_check.py

**功能：** 并发检测所有SOCKS5端口状态并输出汇总报告

**依赖：**
```bash
pip3 install requests[socks]
# 或
apt-get install curl
```

**运行：**
```bash
python3 scripts/quick_socks5_check.py
```

**输出示例：**
```
============================================================
SOCKS5 端口快速检测
服务器: codex.adusun.com
端口范围: 21880-21895
============================================================
✅ 21882 - 175.176.29.168
✅ 21883 - 175.176.28.108
❌ 21884
...
```

### quick_ip.py

**功能：** 快速获取指定SOCKS5代理的出口IP

**运行：**
```bash
python3 scripts/quick_ip.py 21883
```

**输出：**
```
端口 21883 出口IP: 175.176.28.108
```

## 📈 监控最佳实践

### 检测频率建议

- **开发调试：** 按需检测
- **系统测试：** 每小时一次
- **生产运行：** 每15分钟一次
- **故障排查：** 每5分钟一次

### 报告存档规则

1. **日报：** 每天生成一份完整报告
2. **保留期：** 30天内的报告全部保留
3. **归档：** 30天以上移到 `archive/socks5-old-reports/`
4. **命名：** 严格按照 `YYYY-MM-DD_类型.md` 格式

### 告警触发条件

- 在线率 < 50%（9台以上离线）
- 关键端口离线（21883最稳定端口）
- IP地址频繁变化（1小时内变化3次以上）
- 菲律宾卡全部离线

## 🔧 故障处理流程

### 端口不可用排查步骤

1. **确认网络连通**
   ```bash
   ping codex.adusun.com
   ```

2. **测试SOCKS5认证**
   ```bash
   curl -x socks5://ppp:ppp@154.92.2.59:21883 https://ip.sb
   ```

3. **检查设备状态**
   - 登录MQTT平台查看设备在线状态
   - 查看设备上报的网络状态
   - 检查SIM卡注册信息

4. **查看设备日志**
   - SSH到具体设备（通过frp映射）
   - 查看网络模块日志
   - 检查SIM卡信号强度

5. **重启恢复**
   - 通过MQTT发送重启命令
   - 或通过frp SSH手动重启
