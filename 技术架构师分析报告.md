## 2. UDP打洞技术深度分析

### 2.1 技术可行性

**在嵌入式Linux + 移动网络环境下的挑战：**

| 挑战 | 影响 | 解决方案 |
|------|------|---------|
| **CGNAT（运营商级NAT）** | 打洞成功率低 | 需要TURN服务器兜底 |
| **对称NAT** | 几乎无法打洞 | 自动降级到中转模式 |
| **动态IP变化** | 连接频繁断开 | 快速重连机制 |
| **移动网络不稳定** | 丢包率高 | 可靠传输层（KCP/QUIC） |
| **NAT映射超时** | 需要频繁保活 | 心跳包（每30秒） |

### 2.2 成功率预估

根据行业经验：

- **锥形NAT（Full Cone / Restricted Cone）：** 90%+ 成功率 ✅
- **端口限制锥形NAT（Port Restricted Cone）：** 70-80% 成功率 ⚠️
- **对称NAT（Symmetric NAT）：** <10% 成功率 ❌

**菲律宾运营商NAT类型分布（估算）：**
- Globe/Smart 4G/5G：大部分是CGNAT + 对称NAT
- **预估整体打洞成功率：30-50%**

### 2.3 开源方案推荐

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **libp2p** | 成熟、功能完整 | 依赖多、体积大 | ⭐⭐⭐ |
| **WebRTC DataChannel** | 打洞能力强 | 复杂度高 | ⭐⭐⭐⭐ |
| **ZeroTier/Tailscale** | 开箱即用 | 依赖第三方服务 | ⭐⭐⭐⭐⭐ |
| **自研UDP打洞** | 轻量可控 | 开发周期长 | ⭐⭐ |

**推荐：ZeroTier SDK**
- 理由：开源、轻量、打洞能力强、有ARM支持
- 集成难度：中等（5-7天）
- 但仍需要30+天完整测试

### 4.2 路由策略

**正常情况（手机卡在线）：**
```bash
# 默认路由走手机卡
ip route add default via <4G网关> dev usb0 metric 100

# RJ45仅用于管理流量（SSH端口22）
ip rule add from <RJ45网段> table 100
ip route add default via <RJ45网关> dev eth0 table 100
```

**故障情况（手机卡断网）：**
```bash
# 心跳检测失败后，切换默认路由到RJ45
ip route del default via <4G网关> dev usb0
ip route add default via <RJ45网关> dev eth0 metric 50

# 发送告警到MQTT服务器
```

### 4.3 心跳检测机制

```python
# 伪代码
def network_monitor():
    while True:
        # 1. 检查手机卡网络
        if ping("8.8.8.8", interface="usb0", timeout=5):
            cellular_status = "online"
        else:
            cellular_status = "offline"
            switch_to_rj45()
            send_alert("手机卡断网，已切换到RJ45")
        
        # 2. 检查frpc连接状态
        if frpc_connected():
            tunnel_status = "connected"
        else:
            tunnel_status = "disconnected"
            restart_frpc()
        
        # 3. 上报状态到MQTT
        mqtt_publish("device/status", {
            "cellular": cellular_status,
            "tunnel": tunnel_status,
            "traffic": get_traffic_stats()
        })
        
        sleep(30)  # 每30秒检查一次
```

## 6. 远程调试方案（解决核心痛点）

### 问题：手机卡断网后无法远程排查

### 解决方案：多层次远程访问

**层次1：MQTT命令通道（最可靠）**
```
设备 → MQTT服务器（菲律宾/香港）
- 即使手机卡断网，通过RJ45仍可连接MQTT
- 支持远程命令：重启、切换SIM、查看日志、修改配置
```

**层次2：frpc隧道（正常情况）**
```
管理员 → 中转服务器 → frpc隧道 → 设备SSH
- 手机卡在线时的主要访问方式
```

**层次3：RJ45直连（应急）**
```
现场人员 → RJ45网线 → 设备SSH
- 手机卡断网时的最后手段
- 需要现场有人操作
```
