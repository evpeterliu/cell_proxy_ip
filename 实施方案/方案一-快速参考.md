## 🎯 一句话总结

---

## ✅ 核心功能（30天交付）

1. **双网卡智能路由** - 手机卡 + RJ45共存，自动切换
2. **心跳检测** - 每30秒检测网络状态，自动故障切换
3. **MQTT远程管理** - 远程切换SIM卡、重启设备、查看状态
4. **自动故障诊断** - 自动识别故障原因，生成诊断报告
5. **OTA升级** - 远程升级固件，为后期扩展做准备
6. **完善监控** - Grafana实时监控所有设备状态

---

## 🏗️ 系统架构（简化版）

```
用户 → 香港中转服务器 → frp隧道 → net终端(手机卡) → 目标网站
                                    ↓
                                  MQTT
                                    ↓
                              监控系统
```

**关键组件：**
- **设备端：** tiny-frpc + Python监控脚本 + MQTT客户端
- **服务器端：** frps + gost + Mosquitto + Grafana
- **网络：** 手机卡（主） + RJ45（备用）

## 🎯 解决的核心痛点

### 问题：手机卡断网后无法远程排查

### 解决方案：

1. **MQTT远程命令** - 即使手机卡断网，通过RJ45仍可连接MQTT执行命令
2. **自动诊断** - 自动检测SIM卡、信号、流量、系统状态
3. **远程切换SIM卡** - 20秒内切换到备用SIM卡
4. **RJ45备用通道** - 现场人员可通过网线SSH进入

**故障排查时间：从数小时降低到 <30分钟** ✅

## 🚀 部署步骤

### 1. 准备服务器（1天）

```bash
# 香港中转服务器
docker-compose up -d  # 启动 frps + gost + nginx

# 菲律宾MQTT服务器
docker run -d -p 1883:1883 eclipse-mosquitto
```

### 2. 配置设备（每台15分钟）

```bash
# 1. 安装软件
apt install python3 python3-pip frpc

# 2. 配置frpc
cp frpc.ini /etc/frp/frpc.ini

# 3. 安装监控脚本
cp network_monitor.py /opt/
systemctl enable network-monitor

# 4. 启动服务
systemctl start frpc network-monitor
```

### 3. 验证（10分钟）

```bash
# 检查设备状态
mosquitto_sub -t "device/+/status"

# 测试代理
curl -x socks5://proxy.example.com:1080 https://ipinfo.io
```
