## 1️⃣ 项目背景和目标

### 1.1 业务场景
- **当前状态：** 使用RV1103设备 + EC600M模块提供手机卡IP代理服务
- **部署地点：** 菲律宾（前期10-50台）
- **目标用户：** 需要稳定手机卡IP的业务场景

### 1.2 核心痛点

#### 问题描述
当手机卡断网后，无法远程排查原因：
- ❌ 不知道是流量耗尽、信号问题还是SIM卡故障
- ❌ 需要现场人员到机房才能诊断
- ❌ 故障恢复时间长（数小时）

#### 解决方案
通过 **双网卡（手机卡+RJ45）+ MQTT远程管理** 实现：
- ✅ 手机卡断网时，通过RJ45通道远程诊断
- ✅ 自动故障检测和切换
- ✅ 远程切换SIM卡（20秒内完成）
- ✅ 故障恢复时间降到 <30分钟

### 2.2 关键技术决策

#### 决策1：设备端使用Python
**理由：**
- 快速开发，符合团队现有技术栈
- 网络监控、MQTT客户端有成熟库
- RV1103资源足够（内存512MB-1GB）

**对比Go：**
- Go性能更好，但开发速度慢
- 30天时间紧张，优先考虑开发效率

#### 决策2：服务端MQTT调度用Go + PostgreSQL
**理由：**
- 高性能，适合处理大量设备连接
- 并发能力强
- 部署简单（单一二进制）

#### 决策3：采用中转服务器模式（第一阶段）
**理由：**
- 技术成熟，frps + gost已验证可行
- 开发周期短（20天）
- 带宽成本可接受（$150-750/月，10-50台设备）

### 3.2 技术栈详细说明

#### 设备端（RV1103）
- **操作系统：** Ubuntu 22.04 ARM64
- **隧道服务：** tiny-frpc（反向连接到中转服务器）
- **代理服务：** microsocks（SOCKS5本地代理）
- **监控脚本：** Python 3.11
  - 网络状态监控
  - 心跳检测
  - 自动故障切换
- **MQTT客户端：** paho-mqtt（与调度服务通信）
- **进程管理：** systemd

#### 服务器端
**香港中转服务器：**
- frps：反向代理服务器
- gost：SOCKS5代理转发
- Nginx：负载均衡和SSL终止

**菲律宾MQTT服务器：**
- Mosquitto：MQTT Broker
- Go MQTT调度服务：设备管理、命令分发
- PostgreSQL：设备状态、命令历史、日志存储

**菲律宾监控服务器：**
- Grafana：实时监控面板
- Prometheus：指标采集和存储

### 3.3 关键功能实现

#### 1. 双网卡智能路由

**工作原理：**
```python
# 伪代码示意
while True:
    if ping_test("8.8.8.8", interface="usb0"):  # 手机卡
        # 手机卡正常，使用手机卡
        set_default_route("usb0")
    else:
        # 手机卡故障，切换到RJ45
        set_default_route("eth0")
        notify_mqtt("network_switched", {"from": "usb0", "to": "eth0"})
    
    sleep(30)  # 每30秒检测一次
```

**关键参数：**
- 心跳间隔：30秒
- 失败阈值：连续3次失败才切换
- 切换时间：<5秒

#### 2. MQTT远程管理

**支持的远程命令：**
- `switch_sim` - 切换SIM卡
- `reboot` - 重启设备
- `get_status` - 查询设备状态
- `run_diag` - 运行故障诊断
- `update_config` - 更新配置

**MQTT主题设计：**
```
device/{device_id}/status     # 设备状态上报
device/{device_id}/command    # 命令下发
device/{device_id}/response   # 命令响应
device/{device_id}/log        # 日志上报
```

#### 3. 自动故障诊断

**诊断项目：**
- SIM卡状态（在线/离线/锁定）
- 信号强度（RSSI值）
- 流量使用情况
- 系统资源（CPU/内存/磁盘）
- 网络连通性

**诊断报告示例：**
```json
{
  "timestamp": "2026-06-02T10:00:00Z",
  "device_id": "device_001",
  "diagnosis": {
    "sim_status": "online",
    "signal_strength": -75,
    "traffic_usage": "8.5GB / 10GB",
    "cpu_usage": "45%",
    "memory_usage": "512MB / 1GB",
    "network_test": {
      "4g": "failed",
      "rj45": "success"
    }
  },
  "conclusion": "4G网络故障，已自动切换到RJ45"
}
```
